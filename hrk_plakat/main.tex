\begin{abstract}
\centering
HRK CRM to system klasy CRM zrealizowany dla firmy HRK Payroll Consulting. Jego celem jest cyfryzacja procesów związanych z zarządzaniem bazą klientów oraz dokumentacją umów. Platforma automatyzuje cykl życia kontraktów, proces waloryzacji stawek oraz raportowanie wskaźników KPI. Wyróżniającym elementem jest wbudowany asystent oparty na lokalnym modelu językowym, który umożliwia analizę dokumentów z zachowaniem pełnej poufności danych.
\end{abstract}
\begin{columns}[T]
\begin{column}[T]{0.48\textwidth}

\begin{block}{Wstęp}

W obliczu rosnącej liczby kontraktów, standardowe arkusze kalkulacyjne stają się niewystarczające. System HRK CRM powstał, aby rozwiązać problem rozproszonych danych i ręcznych wyliczeń.

Główne moduły aplikacji obejmują:
\begin{itemize}
    \item \textbf{Zarządzanie Umowami:} Baza kontrahentów z historią współpracy, przypisanymi opiekunami i archiwum dokumentów.
    \item \textbf{Proces Waloryzacji:} Moduł automatyzujący wyliczanie nowych stawek w oparciu o aktualne wskaźniki zewnętrzne (np. CPI z API GUS~BDL) z lokalnym fallbackiem do bazy w przypadku niedostępności serwisu.
    \item \textbf{Alerty i KPI:} Powiadomienia o wygasających umowach (30/60/90~dni), zaległych waloryzacjach i braku kontaktu z klientem (pow. 90~dni). Dashboard prezentuje bieżące wskaźniki rentowności portfela.
\end{itemize}
Uprawnienia konfigurowane są na poziomie klienta i umowy. Dostęp do platformy zapewnia mechanizm logowania jednokrotnego (Active Directory SSO).
\end{block}

\begin{block}{Projekt aplikacji}

Zastosowano architekturę klient-serwer opartą na bezstanowym REST API. Całość uruchamiana jest z wykorzystaniem konteneryzacji \textbf{Docker}, co ułatwia replikację środowisk i wdrożenia. Stack \textbf{Docker Compose} obejmuje trzy niezależne serwisy: główne API, mikroserwis Active Directory do uwierzytelniania oraz Schema Manager zarządzający strukturą bazy danych. Schemat bazy ewoluuje przez migracje \textbf{Alembic}, zachowując spójność między środowiskami. Podział na izolowane serwisy zapewnia odporność na błędy i gotowość do skalowania horyzontalnego.

\begin{itemize}
\item \textbf{Interfejs użytkownika:} Aplikacja webowa w \textbf{React} (TypeScript) ze stylowaniem przez \textbf{Tailwind CSS} i komponentami \textbf{Radix~UI / shadcn}. Stanem serwera zarządza \textbf{TanStack Query}, a globalnym -- \textbf{Redux Toolkit}. Typy żądań i odpowiedzi generowane są automatycznie z~OpenAPI serwera.
\item \textbf{Logika serwera:} API w \textbf{Pythonie} z frameworkiem \textbf{FastAPI} i ORM \textbf{SQLAlchemy}. Kod strukturyzowany jest w warstwy api -> service -> repo -> models. Weryfikowany testami (\texttt{pytest}).
\item \textbf{Składowanie danych:} Relacyjna baza \textbf{PostgreSQL} rozszerzona o \texttt{pgvector} i kolumny \texttt{JSONB} dla danych niestandardowych klienta. Pliki PDF umów przechowywane są w \textbf{MinIO} z szyfrowaniem SSE i dostępem przez presigned URL; podgląd dokumentów w przegladarce realizuje biblioteka \texttt{react-pdf}.
\end{itemize}

\end{block}

\begin{block}{Implementacja}

Najbardziej innowacyjnym elementem systemu jest wbudowany moduł doradczy -- \textit{Advisor}. Ze względu na poufność danych zdecydowano się na model \textbf{Bielik~4.5B~v3.0} udostępniany przez \textbf{OpenRouter} -- polskojęzyczny LLM zoptymalizowany pod kątem rozumienia dokumentów prawnych i kadrowych.

System wykorzystuje architekturę \textbf{Retrieval-Augmented Generation (RAG)}:
\begin{enumerate}
    \item \textbf{Parsowanie:} Umowy PDF są dzielone na paragrafy (chunki ~400 tokenów) przy użyciu biblioteki \texttt{pdfplumber}, z zapisem pozycji tekstu na stronie (\texttt{bbox}).
    \item \textbf{Składowanie wektorów:} Reprezentacje liczbowe (embeddings 768-dim, model \texttt{nomic-embed-text}) trafiają do bazy PostgreSQL (\texttt{pgvector}) z indeksem HNSW.
    \item \textbf{Wyszukiwanie:} Zapytania doradców HR używają wyszukiwania semantycznego do wydobycia najbardziej relewantnych klauzul.
    \item \textbf{Odpowiedź modelu:} Asystent działa w dwóch trybach: szybki retrieval (${\sim}200\,\text{ms}$, bez LLM) lub tryb AI z syntezą przez Bielik~4.5B~v3.0 (3--10\,s).
\end{enumerate}

\begin{block}{Podsumowanie}

HRK CRM łączy wymogi bezpiecznego oprogramowania biznesowego z innowacjami sztucznej inteligencji. Platforma eliminuje monotonne procesy manualne i przyspiesza obrót dokumentami, a asystent RAG skraca czas analizy kontraktów. Alerty generowane są w trzech horyzontach czasowych (30/60/90 dni), a wskaźnik inflacji pobierany jest automatycznie z~API GUS~BDL. Wykorzystanie sprawdzonych technologii otwiera drogę do dalszej rozbudowy o moduły analityczne.

\end{block}
